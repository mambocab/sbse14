Part4: Code Up "Early Termination"

There is no point running the optimizer if all the objectives have stopped improving. So let us define improving.

Note:

  • The follow code is part of your generic optimizer; i.e. it should work for any model.
  • The following code has multiple magic numeric options. So Timm's rule of settings apply (keep those options in some global struct and print them all out after each run so you can be sure what was active when you did something).

When to Stop

At the end of each era, reflect on the current and last era.

  • Initially yourself, say, 4 lives;
  • Each era, lose one life.
  • If improvement last era to this one, add one life.

What does improvement mean? In the case of multiple objectives, the binary domination principle was you are better if you are better on at least one objective and worse on none. To implement this definition of frontier, we need to define better and worse. We saw from the above that sometimes the medians can stay the same but the IQR can be reduced. So better had better know about reducing variance.

Also, some goals we are trying to minimize and some we are trying to maximize so our definition of better had better know about that.

Here is the code you need. It turns out that there are two problems:

  1. For each objective, in this era and last era, report if
    • It is unchanged (which we will call same)
    • or if its median value has gotten better (which depends on whether or not we are minimizing or maximizing things);
    • or if its IQR has gotten better (i.e. the IQR in the new era is less than in the old era).
  2. Using that information, across all objectives, conclude that things are better if at least one objective has improved and none have gotten worse.

Reflective over Changes in ONE Objective

Here's something like the code you need to for problem1 (reflecting on each objective). Note that:

  • (new,old) store the values seen for an objective in the (current,previous) era, respectively.
  • From that cache, we can extract the median and IQR which are the 50% percentile and the (75-25)th percentile, respectively.
  • new and old can report how we want to change a goal (if lessp then lower medians are better).
 class Header:
   ...
   def better(new,old):
     "Reflects over one objective."
     t = The.misc.a12 # 'small effect' threshold; use 0.6
     betterIqr = new.has().iqr < old.has().iqr 
     if new.lessp:
       betterMed = new.has().median < old.has().median ## note1
       same      = a12(old._cache, new._cache)  <= t 
     else:
       betterMed = new.has().median > old.has().median  ## note1
       same      = a12(new._cache, old._cache) <= t
     return betterMed, same, betterIqr

Note that:

  • Note1 depending on how you structure your goals, the comparison sign for the medians might actually be reversed. E.g. if when you normalize values 0..1 that you want to minimize to 1..0 then, paradoxically, the comparison operator on these lines might actually be ">=". But that's a low-level engineering issue.
  • Computing medians can mean looking at a lot of numbers. So do that sort only once per era and cache the result. Alternatively, use less numbers...
  • This code needs access to samples of the objective scores seen in each era. The caching technique discussed last week is one way to grant that access, without consuming all available memory.
  • The above code does not return a Boolean; rather, it generates the information that we will use to solve probl em2. Remember problem2? Reflecting over all the objectives? Well...

Reflective over Changes in ALL Objective

Using the above information, we can define better and worse as follows. Note that:

  • This code calls the above code on line nine.
  • This code uses local functions to define worsed and bettered for each individual objective.
  • Unlike the above, this reflects over all objectives;
 class Model:
   ...
   def better(news,olds):
      "Reflects over N objectives."
      def worsed():
        return  ((same     and not betterIqr) or 
                 (not same and not betterMed))
      def bettered():
        return not same and betterMed
      out = False
      for new,old in zip(news.log.y, olds.log.y): # all objectives 
        betterMed, same, betterIqr = new.better(old) ## see above
        if worsed()  : return False     # never any worsed
        if bettered(): out= out or True # at least one bettered
      return out

By the way, the space of all solutions that are better by the above definition represent the Pareto frontier of a set of candidates. A standard heuristic is that when exploring complex optimizations is to:

  • Generate many solutions
  • Cull anything not on the Pareto frontier
  • Generate more solutions, using information from the items on the frontier.
